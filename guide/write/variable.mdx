테스트 내에서 반복적으로 사용되는 값을 변수로 지정하여 사용할 수 있습니다.

[환경변수(Environment Variable)](/guide/write/env-variable)와 달리, 테스트 실행시에만 유지되는 임시적인 변수입니다.

## 1. 변수 생성하기

Reliv 에디터 **[Test Steps]** 패널 우상단 **[+]** 버튼 > **[Add Variable]** 버튼을 눌러 변수를 추가할 수 있습니다.

원하는 변수의 값을 텍스트로 입력하거나, `{` (중괄호)를 입력하여 다양한 종류의 변수를 사용할 수 있습니다.

- **변수** : 해당 스텝 이전에 정의된 변수의 이름을 입력하여, 변수 값을 사용할 수 있습니다.
- **환경변수** : 프로젝트 전체에 적용되는, 사전에 정의한 환경변수입니다.
- **함수** : 특정한 규칙의 문자열을 생성하여 사용할 수 있습니다.

현재 지원되는 함수들은 다음과 같습니다.
| 함수 | 설명 |
| --- | --- |
| `random()` | 랜덤한 영문/숫자 문자열을 생성합니다. <br /> 기본값으로 길이 10의 문자열을 생성합니다. <br /> `random(1)` 과 같이 생성 문자열의 길이를 지정할 수 있습니다. |
| `now()` | 실행 시의 현재 시간 문자열을 생성합니다. <br /> 기본값으로 ISO 8601 포맷의 현재 시간 문자열을 생성합니다. <br /> `now('YYYY-MM-DD')` 와 같이 [day.js 포맷](https://day.js.org/docs/en/parse/string-format)을 입력하여 양식을 지정할 수 있습니다. |

## 2. 변수 사용하기

텍스트 입력창에서 `{` (중괄호)를 입력하면, 변수/환경변수/함수 목록이 나타납니다.

클릭하여 변수를 삽입할 수 있습니다.

<Note>
  각 액션블록은 고유한 변수목록(namespace)을 가집니다. <br />
  한 액션블록(또는 최상위 레벨)에서 정의한 변수는, 내부에 포함된 액션블록에서 인식되지
  않습니다. <br />
  액션블록 외부의 변수를 액션블록 내부에서 사용하고 싶다면, [액션블록
  파라미터](/guide/action-block#3)를 사용하세요.
</Note>
